# 2026-04-10 — harvest E2E 안정화 + 5개 항목 적용

> harvest 파이프라인의 E2E 테스트를 수정하고, 수집된 외부 트렌드 5개 항목을 하네스에 실제 적용했다.

## 배경

어제 첫 harvest를 돌렸지만 스크립트 버그가 있었고, 수집된 트렌드 항목도 아직 적용되지 않은 상태였다. 오늘은 파이프라인 안정화와 실제 적용을 동시에 진행했다.

## 작업 내역

### harvest E2E 테스트 & 버그 수정

`run-harvest.sh`의 2차 E2E 테스트를 진행하며 발견된 버그를 수정했다. `claude -p --allowedTools` 옵션도 추가해 파이프라인 실행 시 사용 가능한 도구를 명시적으로 제한했다.

### harvest items 1-5 적용

수집된 외부 트렌드 중 5개 항목을 하네스에 실제 적용했다:
1. **Context budget** -- 컨텍스트 예산 관리
2. **Troubleshooting** -- 트러블슈팅 가이드
3. **Parallel safety** -- 병렬 실행 안전성
4. **Branch protection** -- 브랜치 보호 규칙
5. **MCP setup** -- MCP 서버 설정

각 항목마다 별도 커밋으로 관리해 추적성을 확보했다.

## 숫자로 보는 하루

| 항목 | 값 |
|---|---|
| 커밋 | 11 |
| 수정 파일 | 28 |
| 적용된 harvest 항목 | 5 |

## 회고

파이프라인이 "돌아가기만 하는" 상태에서 "안정적으로 돌아가는" 상태로 넘어갔다. 특히 harvest 항목을 실제로 적용하면서, 자기 개선 파이프라인이 이론이 아니라 실제 결과물을 만들어낸다는 것을 체감했다. 5개 항목이 모두 하네스 품질에 직접 기여하는 내용이라 만족스럽다.
