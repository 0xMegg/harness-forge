# 2026-04-08 — 하네스 자기 개선 파이프라인 구축

> harness-forge에 trend-harvester를 도입하고, harness-template을 순수 템플릿으로 분리해 자기 개선 파이프라인의 첫 사이클을 완주했다.

## 배경

하네스 시스템이 프로젝트마다 중복 설정으로 커지고 있었다. 품질을 측정하고 외부 트렌드를 수집해 스스로 개선하는 "자기 개선 파이프라인"이 필요했고, 그 엔진 역할을 harness-forge가, 프로젝트 초기 세팅 역할을 harness-template이 각각 담당하도록 구조를 잡았다.

## 작업 내역

### trend-harvester 파이프라인 추가

harness-forge에 `trend-harvester` 기능을 도입했다. 외부 기술 트렌드를 수집하고, fitness filter로 평가한 뒤, 적합한 항목을 하네스 설정에 반영하는 전체 흐름을 구현했다. README 문서도 함께 작성.

### harness-template 순수 복원

harness-template이 자기 개선 파이프라인까지 품고 있어서 역할이 모호했다. self-improvement pipeline 기능을 harness-forge로 이관하고, 시스템 파일을 분리해 harness-template을 순수한 프로젝트 초기화 템플릿으로 복원했다.

### harness-report.sh --target 옵션

측정 대상을 분리할 수 있도록 `harness-report.sh`에 `--target` 옵션을 추가했다. forge와 template의 점수를 독립적으로 측정할 수 있게 되었다.

### 첫 파이프라인 실행

harvest 파이프라인을 실제로 돌려봤다. 수집 → 평가 → 반영까지 첫 사이클 실행 결과를 커밋에 반영했다. 파이프라인이 실제로 동작한다는 것을 확인한 의미 있는 순간.

## 회고

"자기 자신을 개선하는 시스템"이라는 콘셉트가 구호에서 실제 동작하는 코드가 되었다. 특히 template과 forge의 역할을 명확히 분리한 것이 핵심이었다 -- template은 "새 프로젝트를 어떻게 시작할 것인가", forge는 "기존 프로젝트를 어떻게 개선할 것인가"라는 단일 책임을 갖게 됐다. 첫 harvest 실행이 성공적으로 끝난 만큼, 이제 반복 실행하면서 파이프라인 자체를 다듬어갈 차례다.
