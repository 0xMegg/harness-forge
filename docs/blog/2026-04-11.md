# 2026-04-11 — harness-forge 대규모 자기 개선 배치

> trend-harvester에서 harness-forge로 리네임하고, evaluation loop부터 fitness-filter까지 핵심 파이프라인을 전면 재설계한 날.

## 배경

harness-forge는 외부 트렌드를 수집하고 자체 품질을 측정하는 자기 개선 시스템이다. 기존 trend-harvester라는 이름이 실제 역할을 제대로 반영하지 못한다는 판단 하에 프로젝트명을 harness-forge로 변경했고, 이 기회에 쌓여 있던 구조적 개선을 한꺼번에 밀어넣었다.

## 작업 내역

### harness-report 재설계

기존 리포트 구조가 점수 산출 과정을 투명하게 보여주지 못했다. 평가 축별 가중치와 원점수를 분리하고, 리포트 출력 포맷을 정리해서 한눈에 현재 상태를 파악할 수 있게 만들었다.

### evaluation loop + fitness-filter 보강

자기 개선의 핵심인 evaluation loop를 안정화했다. fitness-filter에 5축 평가 기준을 명확히 정의하고, 외부 트렌드 아이템이 실제로 적용할 가치가 있는지 걸러내는 흐름을 다듬었다.

### run-task / run-epic 개선

태스크 실행 단위와 에픽 실행 단위의 경계를 명확히 했다. push-only workflow를 도입해서 변경사항이 자동으로 반영되는 구조로 전환했다.

### audit-coherence tool

시스템 전체의 일관성을 검증하는 audit-coherence 도구를 추가했다. 설정 파일, 스킬 정의, 문서 간의 불일치를 잡아내는 역할이다.

### harness-template 14회 동기화

템플릿 저장소와의 싱크를 14회 수행. 구조 변경이 크다 보니 템플릿 쪽도 함께 맞춰야 할 부분이 많았다.
